# wDMPNN Stage 2D Baseline — Run Instructions

## Goal

Train wDMPNN on `ea_ip` for all three generalization splits to serve as a
baseline comparison for the Stage 2D architecture-aware models.

## What Gets Produced

Each fold saves a `.npz` file containing:
- `y_true`, `y_pred` — real-scale predictions per test sample
- `test_indices` — row indices into `data/ea_ip.csv`
- `smiles_A`, `smiles_B` — monomer SMILES
- `fracA`, `fracB` — composition fractions
- `poly_type` — architecture label (alternating/block/random)
- `split_type`, `fold`, `n_train`, `n_val`, `n_test`

## Files Created

```
configs/wdmpnn_a_held_out.yaml        # Config reference
configs/wdmpnn_group_disjoint.yaml    # Config reference
configs/wdmpnn_pair_disjoint.yaml     # Config reference
scripts/run_wdmpnn_stage2d_baselines.sh  # PBS job script (or run locally)
scripts/python/run_wdmpnn_generalization.py  # Custom split trainer
```

## How to Run

### Option A: Submit all three as a single PBS job

```bash
qsub scripts/run_wdmpnn_stage2d_baselines.sh
```

Estimated walltime: ~12h for all 3 splits × 2 targets × 5 folds = 30 runs.

### Option B: Submit each split separately

```bash
SPLIT_MODE=a_held_out qsub scripts/run_wdmpnn_stage2d_baselines.sh
SPLIT_MODE=group_disjoint qsub scripts/run_wdmpnn_stage2d_baselines.sh
SPLIT_MODE=pair_disjoint qsub scripts/run_wdmpnn_stage2d_baselines.sh
```

### Option C: Run a single fold locally (for testing)

```bash
# Test group_disjoint, fold 0 only
python scripts/python/run_wdmpnn_generalization.py \
    --split_types group_disjoint \
    --folds 0 \
    --targets "EA vs SHE (eV)"

# Dry run (verify splits only, no training)
python scripts/python/run_wdmpnn_generalization.py --dry_run
```

### Option D: a_held_out only (via train_graph.py)

```bash
python scripts/python/train_graph.py \
    --dataset_name ea_ip \
    --model_name wDMPNN \
    --task_type reg \
    --polymer_type copolymer \
    --split_type a_held_out \
    --save_predictions \
    --target "EA vs SHE (eV)"

python scripts/python/train_graph.py \
    --dataset_name ea_ip \
    --model_name wDMPNN \
    --task_type reg \
    --polymer_type copolymer \
    --split_type a_held_out \
    --save_predictions \
    --target "IP vs SHE (eV)"
```

## Compatibility with Stage 2D Analysis

After training completes, the predictions are directly compatible with:
- `analyze_pair_disjoint_transfer.py` — add wDMPNN to the comparison
- Architecture-deviation R² computation (uses `test_indices` → group_key lookup)
- Distribution matching (uses `poly_type`, `smiles_A`, `smiles_B`)

### To integrate wDMPNN into existing analysis:

1. Load predictions from `predictions/wDMPNN_Gen/`
2. Map `test_indices` to `data/ea_ip.csv` rows
3. Compute group_key = `(smiles_A || smiles_B || fracA || fracB)`
4. Compute architecture-deviation: `Δy = y - mean(y | group_key)`
5. Compare with Stage 2D models in the summary table

## Splits Match Exactly

The group_disjoint and pair_disjoint splits are generated by the **same functions**
(`generate_group_disjoint_splits`, `generate_pair_disjoint_splits`) with the same
seed (42) as used by `run_stage2d_generalization.py`. This guarantees identical
train/val/test partitions for fair comparison with Stage 2D models.
